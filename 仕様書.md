# スケジュール管理システム 仕様書

## 概要
月間・日次のスケジュール管理を行うWebアプリケーション。対応者ごとの作業チケットをカレンダー形式で可視化し、管理する。

## 技術スタック
- **フレームワーク**: React 18
- **ビルドツール**: Vite 6
- **言語**: JavaScript (JSX)
- **バックエンド**: Supabase (PostgreSQL / Auth / Storage)

## 機能一覧

### 1. 月間カレンダービュー
- 月ごとのカレンダーグリッド表示
- 各対応者のカラーヘッダー付きでチケットをグループ表示
- 対応者枠と名前はパステルパープルカラーで統一表示
- 各チケットは2行構成（1行目：タイプ/号機/物件名/同行者、2行目：時間指定/作業内容/エリア＋県別[背景付き]/TIME）
  - ※エリアバッジはエリアごとの色（`getRegionColor`）を背景に適用し、視認性を向上。

  - ※作業内容は人間工学に基づいたディープブルー（#1e40af）かつ適切な太さ（800ウェイト）を適用。黒太字の「ごちゃごちゃ感」を避けつつ、重要項目としての視認性を確保。
  - ※同行者は作業内容の右側に専用アイコン（`IconPeople`）と併せて苗字を表示
  - ※TIME(所要時間)は数値バッジとして表示（`h`表記なし）
- 各日ヘッダーに配置された要素：
  - 日付、当日の総件数、チケット追加ボタン（＋）
  - **未対応バッジ**: 対応者が未指定のチケットがある場合、赤いドットバッジに件数を表示
  - **未稼働者リスト**: 予定のないワーカーを小さなアイコンバッジで一覧表示
- **管理者ダイレクト編集**: 管理者（isAdmin=true）の場合、チケットをクリックすると閲覧モーダルをスキップし、直接編集モーダルを展開
- **未割り当て案件の配置**: 未割り当てのチケットは、対応者ごとの「予定済み」リストの下部に分かりやすくまとめて表示
- **表示レイアウトの最適化**:
  - 「画面幅に合わせる」がオフの場合、1画面に2曜日分を表示するよう固定。
  - メインコンテンツエリアに横スクロール（`overflow-x: auto`）を採用。
- **祝日・日曜の強調**: 祝日・日曜日は日付と背景（薄赤）を赤色で強調表示。
- **カレンダーナビゲーション**: 
  - ミニカレンダーから日付を選択した際、現在の表示モード（月間/日間）を維持したままジャンプ。
  - ミニカレンダー内でも祝日を赤字で表示。
- 対応者名クリックによる「アクションメニュー」展開（休暇設定、予定追加ボタン、別対応者への「チケット一括移動」）
- ドラッグ＆ドロップによるチケット移動・並び替え（全域ドロップ検知、＋マーク表示。日・対応者を跨ぐ移動も可能）
- 管理者のみの対応者（予定なし）については一覧およびフィルター・凡例から自動的に非表示化

### 2. デイリー詳細ビュー
- 対応者別のチケット一覧をテーブル形式で表示
- **レイアウトの最適化 (1画面収容)**:
  - 画面幅に関わらず常に1画面（100%幅）に収まるよう調整。拡大モード（スケール）の設定を無視して表示。
  - **デイリービューのレイアウト刷新と情報集約**:
  - **サマリー情報のヘッダー集約**: 
    - 日付の横に、その日の全訪問エリアを「障害（障害、2次、定期）」と「工事（工事、設置）」に分けて一括表示する機能を追加。
    - ワーカー名の横に、その日の全エリア・同行者・移動情報を集計表示するように改善。

  - **テーブル項目の整理**: カード内の「エリア」「同行者」「移動」を非表示にし、不要な「コース」「対応日」を一覧から削除。
  - **物件名と作業名の大幅拡大**: 各列幅を320pxまで拡大し、物件名は2行、作業内容は3行までの複数行表示を徹底することで一覧性を極限まで向上。
  - **列順序の最適化**: 「結果」列を「備考」の右隣へ移動。
内容ハミ出し時の省略記号（...）表示により、一覧性と整頓された外観を両立。
- **空間の美学と視認性**:
  - ヘッダー（日付表示・追加ボタン）とカードの間、およびカード同士の余白を拡大し、ゆとりのある構成に改善。
  - 各行（`tr`）の高さを一定（72px）に固定。
  - 「作業内容」などの長文セルは最大高さを制限し、セル内スクロールを可能にすることで行の高さの不揃いを解消。
  - **詳細ポップアップ (ItemPopup)**: 「作業内容」セルをクリックすると表示。以下の情報を整理して提供：
  - **タイトル**: タイプ、号機（1行固定）、物件名（最大2行表示で全表示を保証）。ベースライン基準の高度な水平アライメントを適用。
  - **サブ情報**: 日計、時間指定、対応者、同行者、TIME
  - **詳細項目**: 依頼番号、BOX、種別、結果、エリア、県別、移動、コース。
  - **アライメント**: `box-sizing: border-box` を適用した38px固定高のグリッドチップを採用し、情報の有無に関わらず美しい整列を実現。
  - **作業内容**: 全幅かつ拡大枠（min-height 120px）で表示。
  - **備考**: 最下部。
  - **編集制限（ロック機能）**: ポップアップ経由の編集ではデフォルトで「日付」と「対応者」が固定されます。管理者（admin）および配車担当（dispatcher）は南京錠アイコンからロックを解除可能です。
- 各行（対応者グループごとのヘッダー）に、指定ユーザーおよびその日の「＋ 追加」ボタンをインライン設置
- **ステータス別カラーリング**:
  - **完了**: 行全体をグレー、操作セルを濃いグレーで表示
  - **キャンセル**: 行全体を薄赤、操作セルを赤い背景色で表示
- **インライン編集**: セルを直接編集可能（「結果」列以外）
- **操作列の固定**: テーブルを横スクロールしても右端操作列（完了・キャンセル・編集・移動・削除）が常に固定表示されるよう `position: sticky` を適用
- **キャンセル機能**: ❌ボタンで結果を「キャンセル」に設定し、担当者を自動で解除（未定へ移動）

### 3. Processing Poolビュー
- 担当者・日付が未定のチケットを一時保管・管理するビュー
- カード形式の一覧表示、依頼番号/号機/物件名での検索
- 「配置」ボタンから特定の日付・対応者へチケットを割り当て

### 4. ズーム機能と表示スケール
- **デフォルトスケール**: 全体的なフォントサイズおよびレイアウト（列幅等）を従来の110%に底上げし、これを標準（100%）として設定。これによりブラウザズームなしで視認性の高い表示を実現。
- 画面右上のヘッダー部に拡大・縮小コントロールを配置
- 50% 〜 200% の範囲で、画面全体の表示スケールを10%刻みで調整可能
- `transform-origin` を `top left` に設定し、拡大時に左端が隠れる問題を防止

### 5. Supabase 連携
- **Microsoft SSO ログイン**: Azure AD 設定を利用した `supabase.auth.signInWithOAuth` でのセキュアな認証。未ログイン時は専用のサインイン画面にルーティングされ、ログイン後のセッション情報をもとにアプリケーションを提供。
- **外部DB同期**: 対応者リスト（`profiles` テーブル）を起動時に自動取得。`sche_role` が定義されたメンバー（admin/worker/viewer）のみを抽出。また、管理者設定画面からメールアドレスで検索し、存在確認をした上で新規対応者を登録可能。
- **権限管理 (Role-Based Access Control)**: ログインユーザーのメールアドレスと `profiles.sche_role` カラムを照合し、下記権限を動的に判定。

| 役割 (Role) | 役割名 | チケット追加 | チケット編集 | チケット削除 | ロール管理 |
| :--- | :--- | :---: | :--- | :--- | :---: |
| **admin** | 管理者 | ◯ | 全て可能 | 全て可能 | ◯ |
| **dispatcher** | 配車担当 | ◯ | 全て可能 | 全て可能 | ✕ |
| **field_engineer** | 案件担当 (FE) | ◯ | **自分の対応分のみ** | ✕ | ✕ |
| **editor** | 編集専従 | ◯ | **自分が作成した分のみ** | **自分が作成した分のみ** | ✕ |
| **view** | 閲覧者 | ✕ | ✕ | ✕ | ✕ |

- **編集制限（ロック機能）**: デイリービューの詳細ポップアップ経由で編集画面を開いた際、デフォルトで「日付」と「対応者」が固定されます。
  - **権限別制御**: 管理者（admin）および配車担当（dispatcher）は、編集画面内の南京錠アイコンからロックを解除して変更することが可能です。案件担当（FE）や編集専従（editor）はロックを解除できません。
- **Equipment (号機) マスタ管理**: 管理者画面から新規の号機（号機番号・物件名・県別・エリア）を `Equipment` テーブルへ追加可能。管理者のみが利用できるよう RLS で保護。
- **マスタ自動検索**: チケット編集モーダルにて「号機」を入力すると、Supabaseの `Equipment` テーブルを自動参照し、物件名（`Property name`）・県別（`prefectures`）・エリア（`address`）を瞬時に補完。
  - ※テーブル名、カラム名は大文字・小文字およびスペースを厳密に区別して照会を行う（例: `Machine number` をキーに検索）。

### 6. 入力制御・共通UI
- 時間指定フィールドでの全角コロン(：)の誤入力決定時に警告・自動削除
- インライン編集時、ホバーで編集可能領域をハイライト
- **案件タイプ設定のレイアウト修正**: 入力フィールドが外枠を飛び出さないよう、`box-sizing: border-box` を適用してレイアウトを最適化。
- **アイコン・ファビコンの実装**:
  - アプリケーションアイコン（`public/icon.png`）を新規作成。
  - `index.html` に favicon および apple-touch-icon のリンクを追加。
- SVGアイコンによる統一されたビジュアルデザイン

### 7. 初期データの取り込み (Excel)
- **ソース**: 「2026　月間予定表.xlsx」（ユーザー提供）
- **抽出方法**: `extract_to_json.cjs` スクリプトを使用して、1月〜12月の全シートから全スケジュールを抽出。
  - 日付が数値（1-31）として格納されているシートについても、シート名から月を特定して全県・全物件の情報を取得。
- **格納先**: `src/initialTickets.json`

## プロジェクト構造
```
schedule-system/
├── .env                # Supabase APIキー設定
├── index.html          # エントリーHTML
├── package.json        # 依存関係
├── vite.config.js      # Vite設定
└── src/
    ├── main.jsx        # Reactマウントポイント
    ├── supabase.js     # Supabaseクライアント初期化
    └── App.jsx         # メインアプリケーション
```
