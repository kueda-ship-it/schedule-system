# スケジュール管理システム 仕様書

## 概要
月間・日次のスケジュール管理を行うWebアプリケーション。対応者ごとの作業チケットをカレンダー形式で可視化し、管理する。

## 技術スタック
- **フレームワーク**: React 18
- **ビルドツール**: Vite 6
- **言語**: JavaScript (JSX)
- **バックエンド**: Supabase (PostgreSQL / Auth / Storage)

## 機能一覧

### 1. 月間カレンダービュー
- 月ごとのカレンダーグリッド表示
- 各対応者のカラーヘッダー付きでチケットをグループ表示
- 対応者枠と名前はパステルパープルカラーで統一表示
- 各チケットは2行構成（1行目：タイプ/号機/物件名/同行者、2行目：時間指定/作業内容/エリア＋県別[枠付き]/TIME）
  - ※作業内容は人間工学に基づいたディープブルー（#1e40af）かつ適切な太さ（800ウェイト）を適用。黒太字の「ごちゃごちゃ感」を避けつつ、重要項目としての視認性を確保。
  - ※同行者は作業内容の右側に専用アイコン（`IconPeople`）と併せて苗字を表示
  - ※TIME(所要時間)は数値バッジとして表示（`h`表記なし）
- 各日ヘッダーに配置された要素：
  - 日付、当日の総件数、チケット追加ボタン（＋）
  - **未対応バッジ**: 対応者が未指定のチケットがある場合、赤いドットバッジに件数を表示
  - **未稼働者リスト**: 予定のないワーカーを小さなアイコンバッジで一覧表示
- **管理者ダイレクト編集**: 管理者（isAdmin=true）の場合、チケットをクリックすると閲覧モーダルをスキップし、直接編集モーダルを展開
- 対応者名クリックによる「アクションメニュー」展開（休暇設定、予定追加ボタン、別対応者への「チケット一括移動」）
- ドラッグ＆ドロップによるチケット移動・並び替え（全域ドロップ検知、＋マーク表示。日・対応者を跨ぐ移動も可能）
- 管理者のみの対応者（予定なし）については一覧およびフィルター・凡例から自動的に非表示化

### 2. デイリー詳細ビュー
- 対応者別のチケット一覧をテーブル形式で表示
- 各行（対応者グループごとのヘッダー）に、指定ユーザーおよびその日の「＋ 追加」ボタンをインライン設置
- **ステータス別カラーリング**:
  - **完了**: 行全体をグレー、操作セルを濃いグレーで表示
  - **キャンセル**: 行全体を薄赤、操作セルを赤い背景色で表示
- **インライン編集**: セルを直接編集可能（「結果」列以外）
- **操作列の固定**: テーブルを横スクロールしても右端操作列（完了・キャンセル・編集・移動・削除）が常に固定表示されるよう `position: sticky` を適用
- **キャンセル機能**: ❌ボタンで結果を「キャンセル」に設定し、担当者を自動で解除（未定へ移動）

### 3. Processing Poolビュー
- 担当者・日付が未定のチケットを一時保管・管理するビュー
- カード形式の一覧表示、依頼番号/号機/物件名での検索
- 「配置」ボタンから特定の日付・対応者へチケットを割り当て

### 4. ズーム機能と表示スケール
- **デフォルトスケール**: 全体的なフォントサイズおよびレイアウト（列幅等）を従来の110%に底上げし、これを標準（100%）として設定。これによりブラウザズームなしで視認性の高い表示を実現。
- 画面右上のヘッダー部に拡大・縮小コントロールを配置
- 50% 〜 200% の範囲で、画面全体の表示スケールを10%刻みで調整可能
- `transform-origin` を `top left` に設定し、拡大時に左端が隠れる問題を防止

### 5. Supabase 連携
- **Microsoft SSO ログイン**: Azure AD 設定を利用した `supabase.auth.signInWithOAuth` でのセキュアな認証。未ログイン時は専用のサインイン画面にルーティングされ、ログイン後のセッション情報をもとにアプリケーションを提供。
- **外部DB同期**: 対応者リスト（`profiles` テーブル）を起動時に自動取得。`sche_role` が定義されたメンバー（admin/worker/viewer）のみを抽出。また、管理者設定画面からメールアドレスで検索し、存在確認をした上で新規対応者を登録可能。
- **権限管理**: ログインユーザーのメールアドレスと `profiles.sche_role` カラムを照合し、管理者（`admin`）権限（管理者設定へのアクセス等）を動的判定。
- **Equipment (号機) マスタ管理**: 管理者画面から新規の号機（号機番号・物件名・県別・エリア）を `Equipment` テーブルへ追加可能。管理者のみが利用できるよう RLS で保護。
- **マスタ自動検索**: チケット編集モーダルにて「号機」を入力すると、Supabaseの `Equipment` テーブルを自動参照し、物件名（`Property name`）・県別（`prefectures`）・エリア（`address`）を瞬時に補完。
  - ※テーブル名、カラム名は大文字・小文字およびスペースを厳密に区別して照会を行う（例: `Machine number` をキーに検索）。

### 6. 入力制御・共通UI
- 時間指定フィールドでの全角コロン(：)の誤入力決定時に警告・自動削除
- インライン編集時、ホバーで編集可能領域をハイライト
- SVGアイコンによる統一されたビジュアルデザイン

### 7. 初期データの取り込み (Excel)
- **ソース**: 「2026　月間予定表.xlsx」（ユーザー提供）
- **抽出方法**: `extract_to_json.cjs` スクリプトを使用して、1月〜12月の全シートから全スケジュールを抽出。
  - 日付が数値（1-31）として格納されているシートについても、シート名から月を特定して全県・全物件の情報を取得。
- **格納先**: `src/initialTickets.json`

## プロジェクト構造
```
schedule-system/
├── .env                # Supabase APIキー設定
├── index.html          # エントリーHTML
├── package.json        # 依存関係
├── vite.config.js      # Vite設定
└── src/
    ├── main.jsx        # Reactマウントポイント
    ├── supabase.js     # Supabaseクライアント初期化
    └── App.jsx         # メインアプリケーション
```

## 起動方法
```bash
npm install
npm run dev
```
